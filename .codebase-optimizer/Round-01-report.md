## 发现的问题

### 1. [P0] 页面缺失但被引用
- **维度**: 功能完整性
- **文件**: `pages/index/users.uvue:138`
- **描述**: 路由引用 `routeGoto('/pages/user/create')` 但目标文件 `pages/user/create.uvue` 不存在。点击"创建用户"按钮会导致导航失败/崩溃。
- **建议**: 创建 `pages/user/create.uvue` 页面或移除该路由引用。

### 2. [P0] 用户编辑页为空模板
- **维度**: 功能完整性
- **文件**: `pages/user/edit.uvue`
- **描述**: 页面存在但仅包含29行代码，只有 `<template>` 结构和基本 onLoad，没有实际的编辑表单、保存逻辑和提交处理。
- **建议**: 添加完整的表单UI和表单处理逻辑。

### 3. [P0] 设置页面无点击事件处理
- **维度**: 功能完整性
- **文件**: `pages/setting/index.uvue`
- **描述**: 所有菜单项（亮度、音量、时间、网络等）仅显示文本和箭头图标，没有任何 `@tap` 或 `@click` 事件处理器。唯一可交互的是暗黑模式开关。
- **建议**: 为所有菜单项添加点击处理器，导航到相应页面。

### 4. [P0] 播放器页面完全硬编码
- **维度**: 功能完整性
- **文件**: `pages/player/index.uvue`
- **描述**: 页面内容完全静态，设备名硬编码为 "Virtual-NS2K1"，节目名硬编码为 "20250623002"、"20250623003"，图片路径硬编码。没有动态数据加载，没有 API 调用，没有状态管理。
- **建议**: 添加真实数据加载逻辑，移除硬编码内容。

### 5. [P0] 代码重复 - formatDuration
- **维度**: 代码质量
- **文件**: `utils/index.uts:25-32`, `pages/programs/shared/form-utils/index.uts:86-93`
- **描述**: `formatDuration` 函数在两个位置完全重复实现，逻辑相同，仅字符串转换方式略有差异。增加了维护成本。
- **建议**: 将其中一个实现移至 `utils/index.uts` 作为唯一导出，更新所有引用。

### 6. [P0] 代码重复 - getProgramFolder
- **维度**: 代码质量
- **文件**: `pages/programs/shared/media-config/index.uts:6-8`, `pages/programs/shared/form-utils/index.uts:68-70`
- **描述**: 获取程序文件夹路径的函数重复定义。一个使用命名常量 `PROGRAM_FOLDER_KEY`，另一个使用硬编码字符串 `"program-folder"`。可能导致状态不同步。
- **建议**: 统一使用命名常量，确保两个实现返回相同的存储键。

### 7. [P0] BASE_URL 硬编码且重复
- **维度**: 代码质量 / 安全
- **文件**: `api/config.uts:1`, `utils/http/index.uts:1`
- **描述**: API 基础 URL 硬编码为 `http://1.255.226.145:12106`，且在两个文件中重复定义。修改服务器地址需要更新两个位置。
- **建议**: 将 BASE_URL 定义为环境变量，通过构建工具注入。

### 8. [P1] JSON.parse(JSON.stringify(err)) 反模式
- **维度**: 代码质量
- **文件**: 18处（主要在 `pages/devices/*.uvue` 和 `components/timer-controls/*.uvue`）
- **描述**: 在错误处理中频繁使用 `JSON.parse(JSON.stringify(err))` 进行深度克隆。这是 JavaScript 反模式，效率低下且可能导致循环引用丢失或类型丢失。
- **建议**: 使用类型安全的深度克隆方法，或重构错误处理逻辑。

### 9. [P1] 魔法数字散落各处
- **维度**: 代码质量
- **文件**: 多处
  - `pages/programs/edit.uvue:116` - 3000ms (toast 持续时间)
  - `pages/setting/index.uvue:212` - 0.8 (暗黑模式开关缩放)
  - `components/picker-date/picker-date.uvue:63,68` - 2030, 2025 (日期范围边界)
  - `utils/index.uts:42` - 3600 (秒/小时常量)
  - `pages/devices/edittime.uvue:292,316-317` - 60*60000 (时区偏移计算)
- **描述**: 大量原始数字散落在代码中，缺乏语义，不易维护和调整。
- **建议**: 将魔法数字提取为命名常量。

### 10. [P1] 跨层依赖 - router 导入 store
- **维度**: 架构
- **文件**: `common/router.uts:1`
- **描述**: 路由模块直接导入 store 模块 `import { useAuth } from '@/store/index.uts'`，破坏了分层架构。路由逻辑与认证状态紧密耦合。
- **建议**: 通过参数传递或中间件解耦，让路由不直接依赖 store。

### 11. [P1] 页面直接使用 store 而非通过 API 层
- **维度**: 架构
- **文件**: `pages/programs/index.uvue:148`, `pages/programs/editor.uvue:216`
- **描述**: 页面组件直接 import 并使用 store (`import { useState } from '@/store/index.uts'`)，绕过了 API 层，导致数据流不清晰。
- **建议**: 通过 API 层获取数据，页面只负责展示和用户交互。

### 12. [P2] 代码重复 - deepClone
- **维度**: 代码质量
- **文件**: `utils/index.uts:91-99`, `pages/programs/shared/history.uts:85-87`
- **描述**: 深度克隆函数的两种不同实现（递归实现 vs JSON序列化实现），用于相同目的。
- **建议**: 统一使用 JSON 序列化版本（已在其他18处使用），删除递归实现。
